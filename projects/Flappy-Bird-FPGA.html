Flappy Bird
on FPGA

Designed and implemented a simplified retro Flappy Bird game in VHDL, combining finite-state machines, obstacle generation, collision detection, VGA graphics, and score display on FPGA hardware.

Overview

This academic project focused on implementing a simplified Flappy Bird game directly on FPGA using VHDL. The design combines control logic, timing, graphics generation, and hardware-oriented resource management.

The system was structured around several dedicated modules: button pulse detection, bird motion, obstacle positioning, collision detection, VGA rendering, and score display. The final design was validated through simulation, synthesis, and timing analysis.

Project Details

Core Architecture

  • Debounced push-button pulse detection
  • Bird motion controlled by FSM, gravity, and jump impulses
  • X obstacle scrolling and fixed Y openings
  • Collision detection between bird and pipes
  • VGA rendering for gameplay and game states
  • 4-digit 7-segment score display

Timing & Clocking

The FPGA system clock runs at 100 MHz. A clock divider was implemented to generate a slower 50 Hz signal so that bird and obstacle motion remain visually natural and playable.

Gameplay Logic

  • Bird FSM states: INIT, FLY, STOPPED, GAME_OVER
  • Jump handled through temporary upward velocity
  • Gravity continuously pulls the bird downward
  • Obstacles scroll from right to left
  • Collision or ground contact ends the game

Display System

  • VGA synchronization and pixel address generation
  • Bird, obstacles, and text rendering
  • ROM-based “START” and “GAME OVER” screens
  • BCD conversion for score display up to 999
  • Multiplexed 4-digit 7-segment output

Resource Usage

Main Synthesis Results

  • LUTs: 869 used
  • LUTRAM: 3 used
  • Flip-Flops: 638 used
  • BRAM: 60 / 135 used
  • IO: 30 / 210 used

Interpretation

Logic utilization stayed low overall, leaving room for future extensions. BRAM usage was the main cost because the design stores graphical data for VGA screens and game visuals.

Selected Module Costs

  • Pulse detector: 53 LUTs, 54 Flip-Flops
  • Bird module: 112 LUTs, 45 Flip-Flops
  • X obstacle module: 345 LUTs, 85 Flip-Flops
  • VGA module: 406 LUTs, 328 Flip-Flops, 60 BRAM
  • Score display: 9 LUTs, 18 Flip-Flops

Timing Validation

The timing report confirmed that all user-defined timing constraints were met, supporting stable execution of VGA control, obstacle motion, and synchronized gameplay on hardware.
